# 6502 dialect

What the Asm198x 6502 assembler accepts today. The **goal** is
source-compatibility with [ca65](https://cc65.github.io/doc/ca65.html) (the
cc65 suite's assembler), so that existing 6502 source assembles unchanged — see
the [syntax-stance decision](https://github.com/asm198x/asm198x/blob/main/decisions/syntax-stance.md).
That goal is **not yet reached**; this page documents the current subset
honestly, gaps included.

```sh
asm198x program.s -o program.bin
```

The assembler is two-pass: pass one assigns addresses to labels, pass two emits
bytes with labels resolved. Output is a flat binary, contiguous from the first
`.org` (or from `$0000` if none is given).

## Instructions

The full documented 6502 instruction set — 56 mnemonics — is supported, with
every documented addressing mode for each:

- **Load/store:** `LDA LDX LDY STA STX STY`
- **Transfer:** `TAX TAY TXA TYA TSX TXS`
- **Stack:** `PHA PLA PHP PLP`
- **Increment/decrement:** `INC DEC INX INY DEX DEY`
- **Arithmetic & logic:** `ADC SBC AND ORA EOR CMP CPX CPY BIT`
- **Shift/rotate:** `ASL LSR ROL ROR`
- **Jump/call/return:** `JMP JSR RTS RTI BRK`
- **Branch:** `BCC BCS BEQ BMI BNE BPL BVC BVS`
- **Flags:** `CLC SEC CLI SEI CLD SED CLV`
- **`NOP`**

Undocumented ("illegal") opcodes are not yet included.

Mnemonics are case-insensitive (`lda` and `LDA` are the same). Labels are
case-sensitive.

## Addressing modes

The assembler chooses the mode from the operand syntax. The `mode` column is the
label used in the [ISA spec](../isa-spec-format.md#the-mode-label-contract).

| Syntax | Mode | Example |
|--------|------|---------|
| *(none)* or `A` | implied / accumulator | `RTS`, `ASL A` |
| `#value` | immediate | `LDA #$01` |
| `value` (byte) | zeropage | `LDA $10` |
| `value` (word) | absolute | `LDA $1234` |
| `value,X` | zeropage,x / absolute,x | `STA $10,X`, `LDA $1234,X` |
| `value,Y` | zeropage,y / absolute,y | `LDX $10,Y`, `LDA $1234,Y` |
| `(value)` | indirect | `JMP ($FFFC)` |
| `(value,X)` | (indirect,x) | `LDA ($20,X)` |
| `(value),Y` | (indirect),y | `LDA ($20),Y` |
| `label` on a branch | relative | `BNE loop` |

### Zero-page vs absolute

When an operand could be either, the assembler picks **zero-page** if the
operand is a literal constant that fits in a byte *and* the instruction has a
zero-page form; otherwise **absolute**. The choice keys off the literal, never
off a label's value, so instruction sizes stay stable between the two passes. A
forward label therefore assembles as absolute.

> **Current limitation:** indirect-operand parsing requires tight syntax —
> `($20,X)`, not `( $20 , X )`.

## Values and expressions

| Form | Example | Meaning |
|------|---------|---------|
| Hexadecimal | `$1234` | `$`-prefixed |
| Binary | `%1010` | `%`-prefixed |
| Decimal | `4660` | bare digits |
| Character | `'A'` | the byte value of the character |
| Symbol | `loop` | a label's address |
| Low byte | `#<label` | the low 8 bits of a value |
| High byte | `#>label` | the high 8 bits of a value |

`<` and `>` are most often used to load a 16-bit address into a register a byte
at a time:

```asm
        lda #<message    ; low byte of message's address
        ldx #>message    ; high byte
```

> **Gap:** arithmetic in expressions (`label+1`, `end-start`, `*`, shifts) is
> **not yet supported** — a value is a single literal or symbol, optionally with
> a `<`/`>` byte-select. This is a priority for ca65 compatibility.

## Directives

| Directive | Aliases | Effect |
|-----------|---------|--------|
| `.org addr` | | Set the assembly address. The first `.org` also sets the load origin. |
| `.byte v, …` | `.db` | Emit one byte per value. |
| `.word v, …` | `.dw` | Emit each value as two little-endian bytes. |

```asm
        .org $0200
data:   .byte $01, $02, 'A', %1000
ptr:    .word data, $C000
```

> **Gaps:** segments, `.proc`/`.scope`, `.macro`, `.include`, `.res`, string
> directives, and the rest of ca65's directive set are not yet implemented.

## Labels and comments

- A **label** is an identifier followed by `:` at the start of a statement. It
  may sit alone on a line or precede an instruction:

  ```asm
  loop:   dex
          bne loop
  ```

- A **comment** runs from `;` to end of line.

> **Gaps:** local/scoped labels (`@local`, cheap locals) and `name = value`
> constant definitions are not yet supported.

## Worked example

```asm
; countdown.s — store A into $0400..$0407, then return
        .org $0200

start:  lda #$00        ; value to store
        ldx #$08        ; loop counter
loop:   sta $0400,x     ; $0400 + X
        dex
        bne loop        ; until X wraps to zero
        rts
```

Assembles to 11 bytes at `$0200`:

```
A9 00 A2 08 9D 00 04 CA D0 FA 60
```

## Distance from ca65

The headline gaps before this dialect can claim ca65 compatibility: arithmetic
expressions, the full directive set (segments, scopes, macros, `.include`),
scoped/cheap local labels, `name = value` definitions, and string handling. The
[syntax-stance decision](https://github.com/asm198x/asm198x/blob/main/decisions/syntax-stance.md)
tracks the intent; this page tracks the reality.
